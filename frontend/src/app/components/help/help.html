Documentație integrări

Ghid tehnic pentru administrator. Cheile de acces (ANAF, Brevo) se configurează în fișierul .env de pe server — un utilizator obișnuit nu le poate schimba din interfață. După orice modificare în .env, serverul backend trebuie repornit.

Configurare ANAF — e-Factura

1. Cont de dezvoltator pe portalul ANAF

  1. Mergi pe anaf.ro → Servicii Online → Înregistrare utilizatori → Dezvoltatori aplicații.
  2. Alege „Înregistrare pentru API-uri" și completează formularul (sau folosește contul SPV existent).
  3. Confirmă adresa de email cu codul primit.

2. Înregistrare aplicație OAuth (Client ID / Secret)

  1. Autentifică-te cu „Autentificare utilizator" (user/parolă de dezvoltator, nu certificatul digital).
  2. Din meniu, alege „Editare profil Oauth" → „Gestionare aplicații".
  3. Completează: denumire aplicație, Callback URL (adresa publică a serverului + /anaf/callback), și serviciul E-Factura.
  4. Apasă „Generare Client ID" — salvează imediat Client ID și Client Secret afișate (nu mai apar ulterior).

3. Variabile de mediu (.env, pe server)

ANAF_CLIENT_ID=...
ANAF_CLIENT_SECRET=...
ANAF_REDIRECT_URI=https://domeniul-tau.ro/anaf/callback
ANAF_AUTH_URL=https://logincert.anaf.ro/anaf-oauth2/v1/authorize
ANAF_TOKEN_URL=https://logincert.anaf.ro/anaf-oauth2/v1/token
ANAF_REVOKE_URL=https://logincert.anaf.ro/anaf-oauth2/v1/revoke
ANAF_EFACTURA_ENV=test

⚠ ANAF_EFACTURA_ENV=test trimite facturile în mediul de testare ANAF (fără efecte reale). Schimbă în production doar când ești sigur că vrei să trimiți facturi reale către clienți. Necesită o aplicație OAuth separată, înregistrată cu Callback URL-ul de producție.

4. Conectare din aplicație

  1. Mergi la Integrări din meniu.
  2. Apasă „Conectează ANAF" — te redirecționează către ANAF pentru autentificare cu certificatul digital calificat (tokenul USB trebuie conectat la calculator).
  3. După autentificare, ești redirecționat înapoi și cardul ANAF arată „Conectat".

Token-ul de acces expiră la 90 de zile și se reînnoiește automat cât timp refresh token-ul (365 zile) e valid.

Configurare Email — Brevo

1. Cont Brevo

  1. Creează cont pe brevo.com (fost Sendinblue).
  2. În Senders, Domains & Dedicated IPs, adaugă și verifică adresa de email de la care vei trimite (sau domeniul complet, pentru livrabilitate mai bună).

2. Cheia API

  1. Din contul Brevo: SMTP & API → API Keys → Generate a new API key.
  2. Copiază cheia generată (începe cu xkeysib-) — nu mai poate fi revăzută ulterior.

3. Variabile de mediu (.env, pe server)

BREVO_API_KEY=xkeysib-...
BREVO_SENDER_EMAIL=adresa-verificata@domeniultau.ro
BREVO_SENDER_NAME=VJA Conta

4. Testare din aplicație

  1. Mergi la Integrări din meniu.
  2. Cardul „Email (Brevo)" arată „Configurat" dacă variabilele există.
  3. Introdu o adresă de test și apasă „Trimite test" — confirmă că emailul chiar ajunge.
